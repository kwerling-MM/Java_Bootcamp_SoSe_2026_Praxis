Kontext:
Wir entwickeln Jorth, eine stackbasierte Sprache in Java.
Zahlen, Strings und die Commands "+", "-", ".", "dup" funktionieren bereits.

Aufgabe:
Ergänze folgende Vergleichsoperatoren:

<    >    ==    !=

Verhalten:
- Arbeiten nur auf Integern
- Ergebnis ist 1 (true) oder 0 (false)
- oberstes Element ist rechter Operand

Beispiele:
"3 4 <" → "[1]"
"3 4 >" → "[0]"
"3 3 ==" → "[1]"
"3 4 !=" → "[1]"

Wenn nicht zwei Integer vorhanden sind:
- Werte unverändert zurück auf den Stack legen

Wichtig:
- Bestehende Funktionalität darf nicht verändert werden
- Keine komplette Umstrukturierung

Zusätzliche Aufgabe:
Erstelle Tests für alle Vergleichsoperatoren

Ausgabe:
- vollständige Methode run
- neue Tests
